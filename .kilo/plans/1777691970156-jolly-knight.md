# Plan: Fix Firebase MCP Server Slow Loading

## Problem
The `firebase-mcp-server` configured in antigravity is loading slowly due to:
1. Using `firebase-tools@latest` forces npx to check npm registry on every launch
2. OAuth token failures causing retry loops
3. Node.js cold start latency
4. Repeated network calls to Google APIs during initialization

## Root Cause Location
- Config file: `C:\Users\USER\.gemini\antigravity\mcp_config.json` lines 20-29
- Error logs: `C:\Users\USER\.gemini\antigravity\daemon\ls_ede601efd096f05e.log` showing OAuth errors
- Daemon logs: `C:\Users\USER\.gemini\antigravity\daemon\` showing initialization times

## Solution Steps

### Step 1: Pin Firebase Tools Version
**Why**: Prevents npx from checking npm registry for updates on every launch
- Edit `C:\Users\USER\.gemini\antigravity\mcp_config.json`
- Change line 25 from `"firebase-tools@latest"` to a specific version like `"firebase-tools@13.14.0"`
- This eliminates 2-5 seconds of registry lookup per startup

### Step 2: Pre-cache Firebase Tools
**Why**: Ensures the package is already downloaded, avoiding first-run install
- Run in terminal: `npx -y firebase-tools@13.14.0 mcp`
- This downloads and caches the package in npm cache
- After verifying it works, terminate the process (Ctrl+C)

### Step 3: Fix Firebase Authentication
**Why**: OAuth errors cause retry loops, adding ~10-30 seconds to startup
- Run `firebase login` in a terminal to establish persistent credentials
- This creates/updates `C:\Users\USER\.config\configstore\firebase-tools.json`
- Ensures valid tokens are available for API calls
- The logs show: "Failed to get OAuth token: state syncing error: key not found" - indicates corrupted or missing auth state

### Step 4: Verify MCP Server Startup
**Why**: Confirm the fixes reduce initialization time
- Restart antigravity/IDE to trigger MCP server initialization
- Monitor `C:\Users\USER\.gemini\antigravity\daemon\` logs
- Check that server initialization time is under 1 second (vs current 421ms-2.3s)
- Verify no more OAuth error messages in logs

### Step 5: Optional - Enable MCP Server Persistence
**Why**: Avoids cold start entirely if antigravity supports persistent MCP connections
- Check if antigravity has a "keep MCP servers alive" setting
- If available, enable persistent connections so servers don't restart on every use

## Expected Results
- Firebase MCP server startup: 2-5 seconds faster
- No OAuth retry delays
- Consistent sub-second initialization after cache warm

## Rollback Plan
If issues occur:
1. Revert `mcp_config.json` to use `firebase-tools@latest`
2. Delete npm cache: `npm cache clean --force`
3. Re-authenticate: `firebase logout` then `firebase login`

## Trade-offs
- Pinning version means missing security updates unless manually updated
- However, using `@latest` is actually worse as it checks for updates constantly
- Recommended: Update version manually every 1-2 months
